Terraform Azure Storage Account — Short Notes
🎯 Goal

Create an Azure Storage Account using Terraform.

📚 Concepts Learned
1️⃣ AzureRM Provider

Terraform uses the AzureRM provider to communicate with Azure APIs.

provider "azurerm" {
  features {}
}
2️⃣ Resource Group Dependency

Storage Account is created inside a Resource Group.

Terraform automatically understands dependencies using references.

Example:

resource_group_name = azurerm_resource_group.storage_rg.name
3️⃣ Storage Account Naming Rules

Azure Storage Account names must be:

globally unique
lowercase only
3–24 characters
no special characters

Example:

tfstorageacctdemo01
📄 Terraform Configuration
terraform {
  required_providers {
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 4.0"
    }
  }
}

provider "azurerm" {
  features {}
}

resource "azurerm_resource_group" "storage_rg" {
  name     = "rg-storage-demo"
  location = "Central India"
}

resource "azurerm_storage_account" "storage_account_demo" {
  name                     = "tfstorageacctdemo01"
  resource_group_name      = azurerm_resource_group.storage_rg.name
  location                 = azurerm_resource_group.storage_rg.location
  account_tier             = "Standard"
  account_replication_type = "LRS"
}
⚙️ Terraform Workflow
Initialize Provider
terraform init

Downloads required providers.

Validate Configuration
terraform validate

Checks syntax and configuration validity.

Preview Changes
terraform plan

Shows what Terraform will create/modify/destroy.

Deploy Infrastructure
terraform apply

Creates resources in Azure.

Destroy Infrastructure
terraform destroy

Deletes managed resources.

Used mostly for:

labs
testing
temporary environments
🧠 Important Terraform Concepts Learned
Terraform State

Terraform tracks created resources using:

terraform.tfstate

State helps Terraform:

track infrastructure
detect changes
avoid recreating existing resources
Idempotency

Running:

terraform apply

multiple times does NOT recreate everything.

Terraform only changes what is required.

In-Place Update vs Replacement
In-Place Update (~)

Example:

LRS → GRS

Terraform modifies existing resource.

Replacement (-/+)

Example:

Changing storage account name

Terraform destroys old resource and creates new one.

🚫 Files Not To Commit

Add .gitignore:

**/.terraform/*
*.tfstate
*.tfstate.backup
*.tfvars

Never commit:

provider binaries
state files
secrets
